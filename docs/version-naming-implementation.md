# 版本取名功能实现说明

## 功能概述
实现了版本取名功能，允许用户为版本添加可选的名称。

## 实现细节

### 1. 数据模型更新
- 在 `Version` 接口中添加了可选的 `name` 字段
- 更新了数据库 schema，将版本表升级到版本 2，添加了 `name` 字段
- 实现了数据库迁移逻辑，支持从版本 1 升级到版本 2

### 2. 状态管理更新
- 更新了 `versionStore` 的 `createVersion` 方法，支持传入版本名称
- 更新了 `updateVersionInPlace` 方法，支持同时更新内容和名称
- 添加了新的 `updateVersionName` 方法，专门用于更新版本名称

### 3. UI 更新
- 在 MainView 中编辑器上方添加了版本名称输入框
- 版本名称输入框只在选择版本时显示
- 输入框支持实时编辑，并在保存时一并保存版本名称

### 4. 版本树展示更新
- 更新了画布渲染器，在版本节点上显示版本名称
- 如果版本有名称，会以粗体显示在节点第一行
- 调整了内容显示逻辑，确保名称和内容都能合理显示

## 测试步骤
1. 创建一个新项目
2. 在编辑器中输入一些内容
3. 在版本名称输入框中输入名称
4. 点击"保存新版本"或"原地保存"
5. 在右侧版本树中查看版本节点，确认名称显示正确
6. 点击不同的版本，确认名称输入框会显示/隐藏和更新当前版本名称

## 注意事项
- 版本名称是可选的，不输入名称不会影响正常使用
- 版本名称在版本树中会以粗体显示在第一行
- 如果名称太长（超过20个字符），会自动截断并添加省略号
- 保存时会一并保存版本名称和内容
